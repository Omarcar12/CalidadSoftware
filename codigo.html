Buenas prácticas de programación

El código en proyectos educativos debe ser claro, comentado y fácil de mantener para que futuros estudiantes o docentes continúen el trabajo sin fricciones.

Estilo y legibilidad

Usar nombres descriptivos, funciones pequeñas y consistencia en el estilo facilita la lectura del código.

Control de versiones

Git permite colaborar y mantener historial. Incluye mensajes claros y ramas para funciones o correcciones.

Comentarios y documentación

Documentar las API, las dependencias y los pasos para ejecutar el proyecto reduce la curva de aprendizaje.

Ejemplo

// Función que duplica un número
function duplicarNumero(numero) {
    return numero * 2;
}
